Data Forge Plus README

By Lannon Khau

Jun 7, 2025 · 4 min read
D ata Forge Plus is a production-grade, AI-powered data platform designed for scalable, cloud-native data cleaning, transformation, and exploration.

🧠 About the Project

Originally conceived to handle 1.3 million rows of financial transaction data, this platform has grown into a dynamic data engine. It enables anyone—from data scientists to non-technical teams—to upload massive CSVs, clean them with AI, and explore insights using intuitive UIs.

🌐 Architecture Overview

Deployed via AWS, the stack includes:

  • ✅ EC2: Flask backend with Gunicorn and Nginx
  • ✅ S3: Dataset upload and object storage
  • ✅ RDS (MySQL): Relational metadata store
  • ✅ Lambda: Serverless AI agents
  • ✅ Secrets Manager: Securely stores API keys
  • ✅ GitHub Actions: CI/CD deployment pipeline

🧹 AI-Powered Data Pipeline

Users upload CSVs via Flask. The system asynchronously runs the cleaning pipeline (using LangChain agents) via AWS Lambda and logs results to RDS. Cleaned datasets are saved in a separate S3 prefix, and users can explore them through a Streamlit EDA interface.

🛠️ Tech Stack

  • Backend: Flask, Gunicorn
  • Frontend: TailwindCSS, Streamlit, Jinja2
  • AI Agents: GPT-4o, LangChain
  • Infra: EC2, S3, Lambda, RDS, Secrets Manager
  • CI/CD: GitHub Actions

📁 Project Layout

  • /Portfolio_V2 → Flask App
  • /utils → AI logic, S3 handler, OpenAI secrets
  • /data_forge_lite → Streamlit interface
  • /.github/workflows → CI scripts

⚙️ Running Locally

git clone https://github.com/LannonTheCannon/Cloudberry_AWS_Bootcamp.git
cd Cloudberry_AWS_Bootcamp/Portfolio_V2
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
flask run

🚧 Known Limitations

Streamlit’s native file upload limit is ~200MB. This is mitigated by routing uploads directly to S3 using pre-signed URLs.
